# Offline / reproducible ROBOT reasoning

The two ontology modules import BFO 2020, CCO 2.1 (merged), and IAO over the network by
default. For **reproducible, air-gapped, or CI reasoning**, an offline import catalog maps
every external and self import to a local copy:

1. **Populate the local imports** (one-time; BFO + CCO from a CommonCoreOntologies clone,
   IAO fetched once):
   ```
   bash qc/setup-offline-imports.sh
   ```
   This writes `qc/imports/{bfo.ttl, cco.ttl, iao.owl}` (gitignored -- third-party,
   ~2.7 MB; regenerated by the script, not committed).

2. **Reason with the catalog** (no network):
   ```
   robot reason --reasoner ELK --catalog qc/import-catalog.xml \
     --input rwx-acq-base.ttl      --output /tmp/reasoned-base.ttl
   robot reason --reasoner ELK --catalog qc/import-catalog.xml \
     --input rwx-acq-transform.ttl --output /tmp/reasoned-transform.ttl
   ```

`import-catalog.xml` also maps the self IRIs (`w3id.org/rockworx/acq`, `/transform`,
`/sysml`) to the local `.ttl` files, so the modules resolve their cross-imports offline too.
Verified 2026-07-19: both modules reason ELK-consistent, fully offline, with this catalog.

## Combined A-Box + T-Box consistency (the SysML-link demo)

```
bash qc/check-combined.sh
```
Reasons the demonstrator A-Box (SysML model instances + the derived program risk/assessment)
against the FULL T-Box (base + transform + bridge + BFO/CCO/IAO). Verified 2026-07-19:
**ELK-consistent, 0 unsatisfiable classes** -- i.e. the tagged model is reasoner-safe against
the whole ontology stack, not just rule-checked at the instance level.
